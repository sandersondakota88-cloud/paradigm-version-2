# Vessel State File (.vsf) — Format Specification**Version:** 0.2.0-draft**Status:** Experimental-----## OverviewThe Vessel State File is a self-describing serialization format for multidimensional state that spans from fully deterministic to probabilistic. The file carries its own coordinate system, constraint definitions, noise parameters, and state data in a structure that is immediately parseable, plottable, and spatially navigable without external schema or execution.The file **is** the coordinate system. Not a description of one.Noise (δ) is a first-class parameter — not a byproduct. It encodes confidence, uncertainty, and observation depth at every point in the state space, enabling the system to bridge deterministic resolution and probabilistic input within a single format.-----## StructureA `.vsf` file consists of two sections: a **header** and a **body**, separated by a blank line.### HeaderEach column in the header defines a **constraint triad**:```dimension_name:value0=0,value1=1,...,valueN=N|min|max```|Component       |Purpose                                       ||----------------|----------------------------------------------||`dimension_name`|Name of the constraint dimension              ||`valueX=N`      |Maps a categorical label to a numeric position||`min`           |Lower bound of the axis                       ||`max`           |Upper bound of the axis                       |**Example header for the loan eligibility domain:**```credit:prime=0,near-prime=1,sub-prime=2|0|2,product:mortgage=0,personal=1,auto=2,business-line=3|0|3,applicant:individual=0,joint=1,business=2,trust=3|0|3,residency:domestic=0,foreign=1,diplomatic=2|0|2,income:under50=0,50to100=1,100to250=2,over250=3|0|3,employment:employed=0,self-employed=1,retired=2,student=3,unemployed=4|0|4```Properties:- The header is a single line of comma-separated triads (commas separate triads at the top level; colons and pipes separate internal components)- Every parser that reads the header understands the full dimensionality, cardinality, label mapping, and axis bounds of the state space- No external schema required- Dimension order in the header defines axis order### BodyEach subsequent line is a **state point** — a coordinate in the space defined by the header, with an associated noise parameter.```sdf,d0,d1,d2,d3,d4,d5,δ,rt,doc```|Field          |Purpose                                                                                                   ||---------------|----------------------------------------------------------------------------------------------------------||`sdf`          |Signed distance field value: `-1` (inside/valid) or `1` (outside/invalid)                                 ||`d0..dN`       |Numeric position in each dimension, as defined by header mappings                                         ||`δ`            |Noise parameter: confidence coefficient for this state point (0.0 = fully resolved, 1.0 = fully uncertain)||Trailing fields|Resolved properties at this coordinate (rate tier, doc level, etc.)                                       |**Example body rows:**```-1,0,1,0,0,1,0,0.00,A·PREFERRED,BASIC1,2,0,2,1,0,4,0.00,DENIED,MAXIMUM-1,0,1,0,0,1,0,0.34,A·PREFERRED,BASIC```The first row: SDF −1 (valid), fully resolved (δ=0.00) — this state was produced by deterministic cascade. No uncertainty.The second row: SDF +1 (denied), fully resolved (δ=0.00) — deterministic denial.The third row: same coordinate as row 1, but δ=0.34 — this state was produced with probabilistic input (e.g., a name field with partial confidence). The position is the same, but the system’s certainty about it is lower.-----## State SlicingA **state slice** is a filtered subset of the body where one or more dimensions are fixed.To slice: select all rows where `dN = K` for a fixed dimension N and value K. The remaining dimensions retain their header triads. The sub-space is still self-describing.Example: slice on `credit = sub-prime` (position 2):```Filter: column d0 == 2Result: all state points where credit is sub-primeRemaining axes: product, applicant, residency, income, employment```The slice is itself a valid `.vsf` body against a reduced header. Slices compose — you can slice a slice.-----## Noise (δ) — Confidence as a Core ParameterNoise is not error. Noise is **depth of observation**.### DefinitionEvery state point carries a noise coefficient δ in the range [0.0, 1.0]:|δ Value  |Meaning                                                |Source                              ||---------|-------------------------------------------------------|------------------------------------||0.00     |Fully resolved — deterministic cascade, no uncertainty |CSS constraint engine               ||0.01–0.30|High confidence — minor probabilistic input involved   |Validated user input, confirmed data||0.31–0.60|Moderate confidence — significant probabilistic factors|Unverified input, partial data      ||0.61–0.99|Low confidence — speculative or incomplete             |Estimated values, branch in progress||1.00     |Fully uncertain — no resolution has occurred           |Uninitialized, placeholder          |### How δ is ComputedThe noise parameter is derived from the resolution path of each dimension:- **Deterministic dimensions** (CSS-resolved): δ contribution = 0.0. The cascade always resolves. No uncertainty.- **Probabilistic dimensions** (JS-navigated, user input): δ contribution is proportional to the ratio of unresolved or unverified dimensions to total dimensions.```δ = (number of probabilistic dimensions with unverified input) / (total dimensions)```For the loan eligibility example with 6 deterministic + 30 name dimensions:- All 6 business dims resolved by CSS: δ contribution = 0.0- Name field partially filled (15 of 30 positions): δ contribution = 15/30 = 0.5- Combined: δ = (0 + 15) / 36 ≈ 0.42As the name field completes and commits, δ shrinks. When all dimensions are resolved, δ reaches 0.0.### Propagation Across LayersWhen a committed row becomes a relativity axis (backup), its δ propagates as context for subsequent computation:```Layer 0: P_0 with δ_0 (raw input)Layer 1: P_1 with δ_1 = f(δ_0, resolution applied)Layer 2: P_2 with δ_2 = f(δ_1, further resolution)...Layer k: P_k with δ_k → approaches 0.0 as resolution accumulates```Each layer inherits the noise from the previous. The system becomes more confident over time — not because it’s guessing better, but because more state has been deterministically resolved and committed.### δ as Dimensional AnchorIn classical systems, time is the reference frame — “what happened first” determines state. In the VSF model, noise replaces time as the ordering principle:- A point with δ=0.0 is a **hard anchor** — a fixed reference frame in the geometry. Reliable for distance calculations, rollback, and navigation.- A point with δ=0.5 is a **soft anchor** — useful for approximation but subject to revision when more resolution occurs.- A point with δ=1.0 is **unanchored** — it exists in the space but has no observational weight.Distance calculations between two points now carry confidence:```distance(P_a, P_b) = spatial_distance(coords_a, coords_b)confidence(P_a, P_b) = 1.0 - max(δ_a, δ_b)```The less noisy the reference points, the more meaningful the distance between them.### Noise in the HeaderThe noise parameter is declared in the header as a continuous dimension:```δ:noise=continuous|0.0|1.0```This tells any parser: this axis is continuous (not categorical), bounded between 0.0 and 1.0, and represents confidence. It’s self-describing just like every other dimension.### Noise and CompressionNoise enables a fourth compression layer beyond bit-packing, entropy encoding, and spatial coherence:**Layer 4 — Confidence culling**: points with δ above a threshold can be excluded from transmission without losing deterministic state. A client that only needs resolved state can request δ < 0.1. A client that needs the full picture including speculative state requests all rows. The noise parameter becomes a built-in level-of-detail control.### Noise and VisualizationWhen the state space is rendered:- δ = 0.0: solid point, full opacity, sharp edge- δ = 0.5: translucent point, halo/envelope around it representing the uncertainty radius- δ = 1.0: ghost point, barely visible, large haloThe noise halo is not decorative — it represents the actual confidence envelope of the observation. Zooming in (increasing dimensional resolution) shrinks halos. Zooming out merges them. The visual fidelity tracks the computational fidelity.-----## Binary EncodingFor network transmission, the body can be bit-packed using the axis bounds from the header.Each dimension value requires `ceil(log2(max - min + 1))` bits:|Dimension          |Cardinality|Bits required||-------------------|-----------|-------------||credit             |3          |2            ||product            |4          |2            ||applicant          |4          |2            ||residency          |3          |2            ||income             |4          |2            ||employment         |5          |3            ||**Total per point**|           |**13 bits**  |One complete state coordinate in **13 bits** — under 2 bytes.Full 2,880-state space: **5,760 bytes** (~5.6 KB) in binary.Same data as text CSV: **~35 KB**.Compression ratio: **~6:1** before any further encoding.With SDF and resolved properties included, each row grows but remains highly compressible due to low entropy (the SDF field is binary, rate tiers have few unique values).### Delta EncodingFor state transitions (streaming), only the changed dimensions need transmission:```Δ: dimension_index, new_position```A single state change is **2 values** — typically 1 byte. The receiver applies the delta, and the constraint geometry resolves everything else locally.-----## Codec ModelThe `.vsf` format maps directly to media streaming concepts:|Media Concept            |Vessel Equivalent                                    ||-------------------------|-----------------------------------------------------||**Codec**                |Header (defines how to decode the stream)            ||**Keyframe (I-frame)**   |Full state point (all dimensions specified)          ||**Delta frame (P-frame)**|State transition (only changed dimensions)           ||**Buffering**            |Pre-resolving adjacent states in the geometry        ||**Playback**             |Hydrating from serialized state into live application||**Seeking**              |Jumping to an arbitrary coordinate in the state space||**Resolution**           |Dimensional cardinality (add values = higher res)    ||**Signal/Noise**         |δ parameter (confidence level of the stream)         |The browser becomes the **decoder**. The CSS cascade is the **rendering engine**. The DOM is the **display surface**.### Streaming Protocol1. **Initial load**: transmit header + one keyframe (current state coordinate)1. **Interaction**: user changes a dimension → transmit delta frame (1 byte)1. **Buffering**: client pre-resolves adjacent states using shadow probe1. **Resume**: transmit last keyframe → client hydrates to exact state1. **Offline**: entire state space fits in ~6KB — cache the whole geometry-----## Compression for Network TraversalThree layers of compression are available:**Layer 1 — Bit packing**: header-derived bounds allow minimal bit widths per dimension. Described above.**Layer 2 — Entropy encoding**: the SDF field and resolved properties have very low entropy (few unique values, predictable distribution). Huffman or arithmetic coding yields significant further compression.**Layer 3 — Spatial coherence**: adjacent points in the state space tend to share SDF values (valid regions are contiguous, denial regions are contiguous). Run-length encoding along any axis exploits this coherence. This is analogous to how video codecs exploit spatial coherence in adjacent pixels.Combined: the full 2,880-state loan eligibility space, including all resolved properties, should compress to **under 2KB** for network transmission. That’s smaller than most favicon files.-----## Properties- **Self-describing**: header contains complete schema, axis bounds, value mappings, and noise definition- **Self-locating**: every cell knows its position in the geometry from its column index and header triad- **Self-referencing**: every point carries its own confidence coefficient, making the system adaptive- **Sliceable**: any dimension can be fixed to produce a valid sub-space- **Composable**: slices can be sliced; spaces can be merged- **Plottable**: any two dimensions can be projected to a 2D visualization directly from the file- **Streamable**: delta encoding enables real-time state synchronization at ~1 byte per transition- **Cacheable**: entire state geometries fit in browser cache, service worker storage, or CDN edge nodes- **Deterministic**: same file, same coordinate → same resolved state, always- **Confidence-aware**: noise parameter enables probabilistic input without compromising deterministic guarantees- **Self-resolving**: system noise decreases over time as more state commits, approaching full determinism-----## File Extension`.vsf` — Vessel State File## MIME Type (proposed)`application/vnd.vessel.state+csv`-----## LicenseMIT
