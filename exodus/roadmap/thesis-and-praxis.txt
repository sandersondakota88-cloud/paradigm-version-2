VSF / EXODUS — Project MapCompiled from the full project documentation plus the Frame-Mediator, Distributed-Collapse-Network, and discovery-record additions.1. The ThesisOne line: The file is the coordinate system; the cascade is the runtime; δ is the measure.Six anchor claims (in order of evidential strength):	1.	The CSS cascade is a parallel constraint satisfier shipping in every browser today.	2.	δ = unresolved_dims / total_dims is a computable quantity, not a metaphor.	3.	Execute-once-seal-state conserves resolved work; recomputation is waste.	4.	The IPC is the observer — the measurement surface where two frames become relatable.	5.	The .vsf file is a machine: program, proof, and data differentiated only by δ.	6.	Computation is the trajectory from δ=1 to δ=0 within a fixed geometry.Claims 1–3 are existence-proven in working code. Claims 4–6 are formal recognitions of what claims 1–3 already imply. Everything else in the project is either consequence, scaling path, or philosophical framing.2. Core Primitives|Symbol    |Name                 |Definition                                            ||----------|---------------------|------------------------------------------------------||**S**     |State                |Any describable configuration                         ||**M**     |Manifold             |Space of states reachable under rules                 ||**C**     |Compression          |Mapping M → R                                         ||**E**     |Expansion            |Future trajectories accessible from R                 ||**H**     |Entropy              |Standard `log W`                                      ||**δ**     |Unresolved state     |`unresolved_dims / total_dims` ∈ [0, 1]               ||**SDF**   |Signed distance field|Binary `−1/+1` in current impl; continuous on GPU path||**Tetrad**|Structural minimum   |Send / Receive / Potential / Reference                |δ arithmetic for relational systems:δ(A→B) ≠ δ(B→A)                        asymmetric by directionδ_IPC  = √(δ(A→B) · δ(B→A))            symmetric (geometric mean)δ_IPC  = α·δ(A→B) + (1−α)·δ(B→A)       weighted (α = authority balance)α ≈ 0.8 in current PoC (server-authoritative).3. The narrow version — new application architecture, working implementation, migration tool, GPU scaling path — is defensible and publishable. The wide version — computation as geometry, observer as channel, δ as universal primitive — is probably partially right but earns the right to be heard only after the narrow version lands.The six anchor claims at the top of this document are the test. If a new idea doesn’t reduce to one of them in working code within a reasonable horizon, it’s not architecture yet — it’s conjecture. Both are fine; they just shouldn’t be confused.End of map.
