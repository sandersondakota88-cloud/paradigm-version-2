PHASE B - GPU EQUIVALENCE HARNESS

walks every generated constraint set through wgsl, compares per-set hash to the js oracle
Loads tests/gpu-fixtures.json (generated by tests/generate-gpu-fixtures.js). Each fixture carries: - the compiled u32 instruction buffer (hex) - the expected FNV-1a 32-bit hash of the JS-oracle output, packed in (sdf:i32, rth:u32, rt:u32, doc:u32, reg:u32, deny:u32) order per coord For each fixture this page: 1. Uploads the instructions buffer to the GPU 2. Dispatches resolve.wgsl over all 2880 coords 3. Reads back outputs, hashes them with the same FNV-1a, compares Per-set agreement is reported; final summary lists totals and any failures. Serve via HTTP -- fetch from file:// is blocked. One-liner: python -m http.server 8080 (from the canonical-implementation/ directory) Then open: http://localhost:8080/tests/gpu-equivalence.html
expected runtime ~5-60s depending on adapter. one device, one pipeline, fixtures reuse the same buffers.
