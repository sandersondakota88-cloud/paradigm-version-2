STRATIFIED EXECUTION HARNESS

css cascade + js stack machine + wgsl compute shader, concurrent per cycle, coherence-checked
For each fixture in tests/gpu-fixtures.json this harness: 1. Compiles the rule set to CSS, installs as a stylesheet, sets data-attrs on 2,880 probe elements, reads --sdf/--rt/--rth/--doc/--reg/--deny via getComputedStyle on each probe. (this is the real browser cascade) 2. Walks the same compiled u32 instruction stream sequentially in JS, producing index-form output per coord. (this is the JS oracle) 3. Uploads the same u32 instruction stream to a WGSL compute shader and dispatches over all 2,880 coords in parallel. (this is the GPU path) 4. Hashes each path's output bytes with FNV-1a, requires all three hashes to match each other AND the expected hash from the fixtures file. Disagreement at any path fails the cycle. This is NOT three independent runs that get diffed. It is one stratified cycle whose output -- the "stratified agreement hash" -- exists only when all three substrates have produced byte-identical resolution against the same input. The output proves the cycle's stratification was coherent. The cycle's resolution is the joint product of three substrates running concurrently in one runtime window. If any one disagrees, the cycle has no resolution. Throttle the fixture count below to control wall-clock; the CSS path is the bottleneck (~150ms per fixture for stylesheet recompile + recalc). Serve via HTTP -- file:// blocks fetch. python -m http.server 8080 (from canonical-implementation/) http://localhost:8080/tests/stratified-harness.html
