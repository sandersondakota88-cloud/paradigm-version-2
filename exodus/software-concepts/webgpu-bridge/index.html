algorithm #16 verification: css cascade == js stack machine == wgsl compute shader
This harness compiles 11 {when,then} constraints into a 70-instruction postfix u32 buffer, then resolves all 2,880 coordinates of the loan-eligibility state space through three different execution substrates: 1. CSS oracle -- pure-JS port of constraints.md section 4 2. JS oracle -- plain-JS stack machine (identical semantics to the shader) 3. GPU path -- WGSL compute shader via WebGPU All three must agree byte-for-byte. If they do, the constraint geometry is substrate-independent -- the same rules resolve identically whether run by browser style recalc, a JavaScript interpreter, or a compute shader. If one diverges, the JS oracle is the reference (proven correct by the Node-side test-oracle.js, which ran before this page was built). Serve this directory via a local HTTP server (fetch does not work from file://). A one-liner: python3 -m http.server 8080, then open http://localhost:8080/.
