What you're looking at

    

      JS doesn't decide which operation to run. The cascade does.
      Input enters the state space (data-X attribute on #state).
      The cascade resolves --next-op based on current configuration.
      A small dispatcher reads --next-op, runs the named operation,
      writes results back as data-X attributes. The cascade resolves
      the next round. Branch logic is gone; dispatch is by selector match.
