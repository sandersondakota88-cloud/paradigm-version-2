What this exercises

The IDB adapter is the substrate-equivalent of the in-memory adapter
verified in Node. Same contract; different backing. These tests confirm
the contract is honored on browser-side IDB.

  -- open / upgrade lifecycle (creates object stores)
  -- putMap / getMap round-trip with a real preparative map
  -- listMapHashes returns inserted source hashes
  -- deleteMap removes the entry
  -- appendTrace assigns monotonic opIds
  -- listTraces filters by source and sinceOpId
  -- trace retention cap (MAX_TRACES_PER_SOURCE) enforced
  -- clearAll empties both stores

If all tests pass, the storage layer is sound on this device's IDB
implementation.
